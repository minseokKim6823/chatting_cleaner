# 프로젝트 목적

이 프로젝트는 Spring Boot 기반 카카오톡 채팅 제약 판단 및 제재 처리 서비스다.
일반 상담형 챗봇이 아니라, 정책 위반 판단과 제재 처리의 정확성, 일관성, 보안, 기존 연동 호환성을 우선하는 백엔드 서비스다.

핵심 책임:
- 카카오톡 이벤트와 메시지 수신
- 정책 위반 여부 판단
- 경고, 일시 제한, 영구 제한 결정
- 제재 결과의 응답, 저장, 알림 처리

불명확한 요구사항은 추측으로 확장하지 말고 기존 코드와 문서의 의도를 우선한다.

# 프로젝트 개요

이 프로젝트는 Spring Boot 기반 카카오톡 채팅 제약 판단 및 제재 처리 서비스다.

핵심 목적:
- 카카오톡 이벤트와 메시지를 수신한다
- 정책 위반 여부를 판단한다
- 경고, 일시 제한, 영구 제한을 결정한다
- 제재 결과를 응답, 저장, 알림으로 연결한다

이 프로젝트는 일반 상담형 챗봇이 아니다.
정책 판단의 정확성, 일관성, 보안, 기존 연동 호환성을 우선한다.

# 공통 작업 원칙

- 기존 구조와 규칙을 우선 유지한다
- 추측으로 비즈니스 규칙을 추가하지 않는다
- 작은 수정으로 해결 가능한 문제를 대규모 리팩터링으로 풀지 않는다
- 민감정보, 인증, 외부 연동 포맷은 보수적으로 다룬다
- 버그 수정 시 재현 테스트를 함께 고려한다

# 우선순위

작업 시 다음 순서를 우선한다:
1. 기존 API 및 카카오 연동 호환성 유지
2. 보안 및 민감정보 보호
3. 제약 정책 판단의 정확성
4. 예외 응답의 일관성
5. 테스트 가능성
6. 코드 스타일 및 리팩터링

# 로컬 명령어

- 테스트: `./gradlew test`
- 빌드: `./gradlew clean build`
- 로컬 실행: `./gradlew bootRun`

명령어가 실제 저장소와 다르면 현재 저장소 기준 명령을 우선한다.
